# はじめかた（担当者向け・ターミナル不要）

このプロジェクトは **ブラウザだけ** で開発できます。パソコンへのインストールも、コマンド入力も要りません。
自分の担当機能（例: `feature-a`）を、Claude Code に日本語で話しかけて作っていきます。

---

## 事前に必要なもの（2つだけ）

1. **GitHub アカウント**（このリポジトリのメンバーに招待されていること）
2. **Claude のアカウント**（有料プラン。Claude Code のログインに使います）

---

## 手順（初回）

### 1. Codespace を開く
1. ブラウザで https://github.com/babaz-makar/management を開く
2. 緑色の **「Code」** ボタンをクリック
3. **「Codespaces」** タブ → 下の方の **「…」** から **「New with options」**
4. **Branch** で自分の担当ブランチ（例 `feature-a`）を選ぶ ← ここ重要
5. **「Create codespace」** をクリック

> しばらく待つと、ブラウザの中に VS Code（開発画面）が立ち上がります。
> 裏で自動的に準備（`pnpm install`）が走るので、1〜2分待ってください。

### 2. Claude Code にログイン（初回だけ）
1. 画面左のアイコンから **Claude Code** のパネルを開く
2. **「Sign in」** を押して、自分の Claude アカウントでログイン
3. 一度ログインすれば、その Codespace では次から不要

### 3. 話しかけて開発する
Claude Code のパネルに、日本語で頼むだけです。例えば：

> `packages/feature-a` に、〇〇を入力すると△△を表示する画面を作って

Claude がコードを書き、動作確認までやってくれます。
**書いたものは自動的に自分のブランチ（`feature-a`）に保存（push）されます。** git 操作は不要です。

### 4. プレビューを見る
Claude に「プレビューを見せて」と頼むと開発サーバーが起動し、ブラウザに画面が表示されます。

---

## 終わるとき

- ブラウザのタブを閉じるだけで OK（30分放置で自動停止します）
- 作業が完全に終わったら、GitHub の **Codespaces 一覧**から使い終わった Codespace を **削除**してください
  （停止中でも保存容量を消費するため）

---

## ルール（大事）

- **自分の担当フォルダ（`packages/feature-x`）だけ**をさわる
- **`main` ブランチでは作業しない**（統合はオーナーがまとめて行います）
- 詳しい運用ルールは [../CLAUDE.md](../CLAUDE.md) を参照

---

## 困ったら

- 準備が終わらない／エラーが出る → Codespace を一度削除して、手順1からやり直す
- それでもダメ → オーナーに連絡
